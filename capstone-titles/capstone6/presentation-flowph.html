Decoupled Ingestion and Dimension-Agnostic Storage Strategy

Ingestion and Adapters

Acts as a buffer to absorb structural chaos. Special-purpose adapters ingest and normalize scraped raw files (DBM XLS, PhilGEPS CSV, DPWH Portals, COA PDFs) before database insertion, protecting the core storage schema.

Raw Scraped Inputs (Chaos)
DBM Budget Spreadsheets
DPWH Project PDF Portals
COA Audit Reports and Scans

Resilient Storage Core

Decouples records from rigid government hierarchies using three stable tables (`FLOWS`, `FLOW_DIMENSIONS`, `FLOW_EVIDENCE`). Zero SQL migrations are required when departments re-organize or introduce new reporting metrics.

Unified Storage (Dimension-Agnostic)
flows (Vectors)
flow_dimensions (Key-Value Tags)
flow_evidence (Verification Vault)

Translation and Resolver

The Laravel backend resolves dynamic queries at runtime, translating database rows into standardized node-link structures.

Runtime Query Pivot Aggregates flow volumes and traverses key-value metadata dynamically (e.g. Sector → Agency → Supplier).
D3.js Visualization Feed Feeds clean JSON arrays directly to client-side visualization components, decoupling the frontend from data origins.

Pillar 5: Project Roadmap and Limits
FlowPH Title Defense

Project Scope Boundaries

In Scope

  • Unified 3-table relational schema (Flows, Dimensions, Evidence)
  • Dynamic dimension resolver logic for multi-tier pivot querying
  • Interactive D3.js/SVG Sankey visual exploration engine
  • Ingestion Staging Sandbox and Review dashboard with visual diffs
  • Legacy hash-based HTML scrapers (for low-funded LGU portals)
  • Partner DBM webhook API gates with signature key validation
  • AI Watchdog anomaly detection triggers (pricing and velocity checks)
  • Evidence Vault with local SHA-256 cryptographic hashing

Out of Scope

  • Automated PDF OCR scanned text recognition pipelines
  • Real-time bank clearing hooks or automated disbursement holds
  • Consensus blockchain ledgers and public transaction networks

Execution Roadmap

1
Ingestion and API Gates (Weeks 1-4)

Setup database schemas, synonym mapping, legacy scrapers, partner webhooks, and the staging review sandbox panel.

2
Resolver and Viz (Weeks 5-8)

Build dynamic SQL dimension translation engine and integrate responsive client-side Sankey explorer visualization.

3
Watchdogs and Evidence (Weeks 9-12)

Implement AI anomaly detection rules, evidence hashing upload pipelines, and the AI COA dossier compiler.

Difficulty Rating: High (Capability Confirmed)
Key Challenges: Dynamic SQL join optimization, AI anomaly classification accuracy, and file integrity validation under heavy transactional load.
